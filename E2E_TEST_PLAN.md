# Real-Slide E2E Test Plan

이 문서는 발표자(Presenter)와 참석자(Participant) 간의 실시간 상호작용이 주 핵심인 Real-Slide 서비스의 사용자 시나리오 및 E2E 테스트 항목을 정의합니다.

---

## 🎭 Persona 1: 발표자 (Presenter)
**목표:** 원활한 발표 진행, 참여 유도, 실시간 피드백 관리

### 1.1 세션 생성 및 대기실
- [x] 새로운 세션 생성 (세션 제목 및 정보 입력) (2026-03-20 PASS)
- [x] 생성된 세션의 **Share Code** 확인 및 복사 기능 작동 확인 (2026-03-20 PASS)
- [x] **QR 코드** 다이얼로그가 정상적으로 열리고, 올바른 URL이 포함되어 있는지 확인 (2026-03-20 PASS)

### 1.2 슬라이드 관리 및 제어
- [x] 슬라이드 추가: 일반 텍스트 슬라이드, 투표(Vote), 퀴즈(Quiz) 타입이 정상적으로 저장되는지 확인 (2026-03-20 PASS)
- [x] **슬라이드 전환**: 이전/다음 버튼 클릭 시 현재 슬라이드가 동기화되어 바뀌는지 확인 (2026-03-20 PASS)
- [x] **전체화면 발표 모드**: 전체화면 진입 및 단축키(방향키, ESC) 작동 확인 (2026-03-20 PASS)

### 1.3 실시간 상호작용 관리
- [x] **정답/통계 공개**: 투표 및 퀴즈 슬라이드에서 결과 공개 버튼 클릭 시 실시간으로 상태가 전파되는지 확인 (2026-03-20 PASS)
- [x] **손들기(Hands Up) 알림**: 참석자가 손을 들었을 때 발표자 대시보드에 실시간으로 리스트가 업데이트되는지 확인 (2026-03-20 PASS)
- [x] **댓글 섹션**: 참석자가 남긴 댓글이 발표자 화면에 실시간으로 표시되는지 확인 (2026-03-20 PASS)

---

## 👥 Persona 2: 참석자 (Participant)
**목표:** 발표자의 흐름에 따라 실시간으로 참여하고 의견 전달

### 2.1 세션 진입
- [x] **참여 링크(Share Code)**를 통해 세션에 비밀번호나 별도 로그인 없이 즉시 진입 가능한지 확인 (2026-03-20 PASS)
- [x] 닉네임 설정 및 세션 입장 프로세스 확인 (2026-03-20 PASS)

### 2.2 실시간 동기화 및 참여
- [x] **슬라이드 동기화**: 발표자가 슬라이드를 넘길 때 참석자 화면이 즉시(Real-time) 바뀌는지 확인 (2026-03-20 PASS)
- [x] **투표 참여**: 투표 옵션을 선택하고 제출 시 결과가 실시간으로 반영되는지 확인 (2026-03-20 PASS)
- [x] **퀴즈 풀기**: 정답을 선택하고 제출한 뒤, 발표자가 정답을 공개할 때 피드백(정답/오답 표시)이 오는지 확인 (2026-03-20 PASS)

### 2.3 상호작용 도구
- [x] **손들기 버튼**: 버튼 클릭 시 상태가 바뀌고, 발표자에게 알림이 가는지 확인 (2026-03-20 PASS)
- [x] **댓글 작성**: 텍스트 입력 후 전송 시 다른 익명 사용자나 발표자에게 즉시 보이는지 확인 (2026-03-20 PASS)
- [x] **실시간 차트**: 다른 사람들의 투표 결과가 차트에 동적으로 업데이트되는지 확인 (2026-03-20 PASS)

---

## 🛠 공통 / 시스템 테스트
- [x] **Supabase Realtime 연결**: 네트워크 단절 후 재연결 시 상태 동기회 복구 여부 (2026-03-20 PASS)
- [x] **반응형 디자인**: 모바일(참석자 주 사용)과 데스크톱 가로형(발표자 주 사용)에서 UI가 깨지지 않는지 확인 (2026-03-20 PASS)
- [x] **에러 핸들링**: 존재하지 않는 세션 코드 입력 시의 안내 메시지 확인 (2026-03-20 PASS)

위 시나리오를 바탕으로 **Playwright** 또는 **Cypress**를 사용하여 다중 브라우저 컨텍스트(발표자 탭 1개 vs 참석자 탭 N개) 환경에서의 자동화 테스트 코드를 작성할 예정입니다.
